## Overview

The terminal implements a simulated Unix-like file system stored in React component state. This allows for persistent file operations during a terminal session, including navigation, file creation, and directory management.

## File System Structure

### Visual Hierarchy

```
~ (home directory)
├── about.txt
├── skills.txt
├── contact.txt
├── projects/
│   ├── teamfinder.txt
│   ├── spotify-bot.txt
│   ├── quizapp.txt
│   └── electrade.txt
└── experience/
    ├── gradguide-internship.txt
    └── summary.txt
```

## Directory Structure

### Home Directory (`~`)

The root of the file system, containing personal information files and subdirectories.

**Location**: `~`

**Contents**:
- `about.txt` - Personal introduction
- `skills.txt` - Technical skills
- `contact.txt` - Contact information
- `projects/` - Projects directory
- `experience/` - Professional experience directory

**Example**:
```bash
guest@portfolio:~$ ls
about.txt  projects/  experience/  skills.txt  contact.txt
```

## File System Data Structure

The file system is implemented as a nested JavaScript object in React state:

```typescript
interface FileSystemItem {
  type: 'file' | 'directory';
  content?: string;      // For files
  contents?: {           // For directories
    [key: string]: FileSystemItem;
  };
}

const fileSystem = {
  '~': {
    type: 'directory',
    contents: {
      'about.txt': {
        type: 'file',
        content: 'I am Pranav...'
      },
      'projects': {
        type: 'directory',
        contents: {
          'teamfinder.txt': {
            type: 'file',
            content: 'Team collaboration platform'
          },
          // ... more files
        }
      },
      // ... more items
    }
  }
};
```

## Directory Navigation

### Path Resolution

The terminal supports different path formats:

| Path Type | Format | Example | Description |
|-----------|--------|---------|-------------|
| **Absolute** | `~` or `~/path` | `~/projects` | From home directory |
| **Relative** | `dirname` | `projects` | From current directory |
| **Parent** | `..` | `cd ..` | Go up one level |
| **Home** | (empty) | `cd` | Return to home |

### Navigation Examples

<CodeGroup>
```bash Absolute Path
# From anywhere, go directly to projects
guest@portfolio:~/experience$ cd ~/projects
guest@portfolio:~/projects$
```

```bash Relative Path
# From home, use relative path
guest@portfolio:~$ cd projects
guest@portfolio:~/projects$
```

```bash Parent Directory
# Go up one level
guest@portfolio:~/projects$ cd ..
guest@portfolio:~$
```

```bash Home Directory
# Return to home from anywhere
guest@portfolio:~/experience/subdir$ cd
guest@portfolio:~$
```
</CodeGroup>

### Session-Based Storage

The file system is stored in React component state:

    - Directory creations
    - All modifications

  <Card title="Reset On Page Reload" icon="rotate">

  </Card>

### Initial vs. Modified State

**Initial State** (on page load):
```
~
├── about.txt
├── skills.txt
├── contact.txt
├── projects/
└── experience/
```

**Modified State** (after user operations):
```
~
├── about.txt
├── skills.txt
├── contact.txt
├── projects/
├── experience/
├── notes/           ← Created by user
│   └── todo.txt     ← Created by user
└── newfile.txt      ← Created by user
```

**After page reload**: Returns to Initial State

## File System Limitations

<AccordionGroup>
  <Accordion title="No File Editing">
    Once created, file contents cannot be modified. You can only:
    - Create new files (empty)
    - Read existing files
    - Delete files
  </Accordion>

  <Accordion title="No Directory Deletion">
    The `rm` command only works on files, not directories. There is no `rmdir` or `rm -r` functionality.
  </Accordion>

  <Accordion title="No File Moving/Copying">
    Commands like `mv` (move) and `cp` (copy) are not implemented.
  </Accordion>

  <Accordion title="No Permissions">
    There is no permission system. All operations are allowed (if the command supports it).
  </Accordion>

  <Accordion title="No Symlinks">
    Symbolic links are not supported.
  </Accordion>

  <Accordion title="Session-Only Persistence">
    Changes are lost on page reload. The file system resets to its initial state.
  </Accordion>
</AccordionGroup>

### Path Resolution Function

```typescript Terminal.tsx:92-103
const resolvePath = path => {
  if (path.startsWith('~')) return path;
  if (path.startsWith('/')) return path;
  if (currentDirectory === '~') {
    return path === '..' ? '~' : `~/${path}`;
  }
  return `${currentDirectory}/${path}`;
};
```

### Current Path Getter

```typescript Terminal.tsx:81-90
const getCurrentPath = () => {
  const parts = currentDirectory.split('/').filter(Boolean);
  let current = fileSystem['~'];
  for (const part of parts.slice(1)) {
    if (current.contents && current.contents[part]) {
      current = current.contents[part];
    }
  }
  return current;
};
```
