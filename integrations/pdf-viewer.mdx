## Overview

The portfolio includes two PDF viewer implementations:

1. **PdfWindow** - Simple native PDF viewer for resumes
2. **BooksWindow** - Advanced PDF viewer with book library using react-pdf

Both components provide draggable, resizable windows with dark-themed interfaces.

## PdfWindow Component

**Location:** `app/components/windows/PdfWindow.tsx`

A lightweight PDF viewer that uses the browser's native PDF rendering capabilities.

### Features

- Native browser PDF rendering using `<object>` and `<iframe>` fallback
- Draggable and resizable window
- Minimize and maximize controls
- Full-screen capable
- Download fallback for unsupported browsers

### Usage

```tsx
import { PdfWindow } from '@/app/components/windows/PdfWindow';

<PdfWindow 
  filePath="/fullstack_dev.pdf" 
  onClose={() => setShowPdf(false)} 
/>
```

### Props

```typescript
interface PdfWindowProps {
  onClose: () => void;
  filePath: string; // e.g., '/backend_dev.pdf'
}
```

### Implementation Details

**Location:** `app/components/windows/PdfWindow.tsx:63`

```tsx
<div className="flex-1 bg-white">
  <object data={filePath} type="application/pdf" className="w-full h-full">
    <iframe src={filePath} className="w-full h-full border-none" />
    <div className="p-4 text-sm text-gray-800">
      Your browser cannot display PDFs inline. You can
      <a
        href={filePath}
        className="text-blue-600 underline ml-1"
        target="_blank"
        rel="noreferrer"
      >
        download the file
      </a>
      instead.
    </div>
  </object>
</div>
```

The component uses a three-tier fallback strategy:
1. Try `<object>` tag (preferred)
2. Fall back to `<iframe>`
3. Show download link if both fail

### Available PDF Files

The portfolio includes these resume PDFs:

- `/fullstack_dev.pdf` - Full Stack Developer resume
- `/backend_dev.pdf` - Backend Developer resume
- `/flutter-dev.pdf` - Flutter Developer resume

## Image Handling

Book covers include error handling and fallback:

**Location:** `app/components/windows/BooksWindow.tsx:173`

```tsx
<div className="relative w-24 h-32 rounded-lg shadow-lg overflow-hidden">
  {book.coverImage ? (
    <Image
      src={book.coverImage}
      alt={`${book.title} cover`}
      fill
      className="object-cover transition-transform group-hover:scale-105 duration-300"
      sizes="(max-width: 96px) 100vw, 96px"
      onError={e => {
        const target = e.target as HTMLImageElement;
        target.style.display = 'none';
        target.parentElement?.classList.add('fallback-active');
      }}
    />
  ) : null}
  {/* Fallback icon */}
  <div className="absolute inset-0 w-full h-full bg-gradient-to-b from-gray-700/50 to-gray-900/50 flex items-center justify-center">
    <IconBook size={32} className="text-yellow-400/80" />
  </div>
</div>
```

## Performance Considerations

### PDF.js Worker

The BooksWindow uses a CDN-hosted PDF.js worker to offload PDF parsing to a separate thread:

```tsx
const pdfWorkerUrl = `https://unpkg.com/pdfjs-dist@3.11.174/build/pdf.worker.min.js`;
```

This prevents the main thread from blocking during PDF rendering.

### Lazy Loading

Books are only loaded when selected:

```tsx
{!selectedBook ? (
  <div>{/* Show book library */}</div>
) : (
  <div>{/* Show PDF viewer */}</div>
)}
```

## Custom Scrollbar Styling

Both components use custom scrollbar classes:

```tsx
className="overflow-y-auto custom-scrollbar mobile-hide-scrollbar"
```

These classes should be defined in your global CSS to provide a consistent, themed scrollbar experience.
