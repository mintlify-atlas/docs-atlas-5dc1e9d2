## Overview

The portfolio uses Firebase for backend services, specifically Firestore for real-time database operations. Firebase is initialized with environment variables and provides a singleton database instance.

## Setup

**Location:** `lib/firebase.ts`

The Firebase configuration is minimal and follows best practices for Next.js applications.

### Firebase Configuration

**Location:** `lib/firebase.ts:4`

```typescript
const firebaseConfig = {
```

### Firebase Initialization

**Location:** `lib/firebase.ts:14`

```typescript
import { initializeApp, getApps } from 'firebase/app';
import { getFirestore } from 'firebase/firestore';

// Initialize Firebase
const app = getApps().length === 0 
  ? initializeApp(firebaseConfig) 
  : getApps()[0];

const db = getFirestore(app);

export { db };
```

This initialization pattern ensures Firebase is only initialized once, even with Next.js hot module replacement during development.

## Environment Variables

Add these to your `.env.local` file:

```bash
NEXT_PUBLIC_FIREBASE_API_KEY=your_api_key
NEXT_PUBLIC_FIREBASE_AUTH_DOMAIN=your_project_id.firebaseapp.com
NEXT_PUBLIC_FIREBASE_PROJECT_ID=your_project_id
NEXT_PUBLIC_FIREBASE_STORAGE_BUCKET=your_project_id.appspot.com
NEXT_PUBLIC_FIREBASE_MESSAGING_SENDER_ID=your_messaging_sender_id
NEXT_PUBLIC_FIREBASE_APP_ID=your_app_id
```

### Why NEXT_PUBLIC Prefix?

All Firebase environment variables use the `NEXT_PUBLIC_` prefix because they need to be accessible in the browser for client-side Firebase operations. Next.js only exposes environment variables with this prefix to the client bundle.

## Getting Firebase Credentials

1. **Create a Firebase Project:**
   - Go to [Firebase Console](https://console.firebase.google.com/)
   - Click "Add project" or select an existing project

2. **Register Your Web App:**
   - In Project Settings, scroll to "Your apps"
   - Click the Web icon (`</>`)
   - Register your app with a nickname

3. **Copy Configuration:**
   - Firebase will display your `firebaseConfig` object
   - Copy each value to your `.env.local` file

4. **Enable Firestore:**
   - In the Firebase Console, go to "Firestore Database"
   - Click "Create database"
   - Choose production or test mode

### Real-time Updates

```typescript
import { collection, onSnapshot } from 'firebase/firestore';
import { db } from '@/lib/firebase';
import { useEffect, useState } from 'react';

function useRealtimeData() {
  const [data, setData] = useState<any[]>([]);

  useEffect(() => {
    const unsubscribe = onSnapshot(
      collection(db, 'collectionName'),
      (snapshot) => {
        const items = snapshot.docs.map(doc => ({
          id: doc.id,
          ...doc.data()
        }));
        setData(items);
      },
      (error) => {
        console.error('Error fetching real-time data:', error);
      }
    );

    return () => unsubscribe();
  }, []);

  return data;
}
```

## Firestore Security Rules

Ensure you set up proper security rules in the Firebase Console:

```javascript
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    // Allow read access to all
    match /{document=**} {
      allow read: if true;
    }
    
    // Restrict write access
    match /{document=**} {
      allow write: if request.auth != null;
    }
  }
}
```

Adjust these rules based on your security requirements.

## Firestore Queries

### Query with Filters

```typescript
import { collection, query, where, getDocs } from 'firebase/firestore';
import { db } from '@/lib/firebase';

async function queryData() {
  const q = query(
    collection(db, 'collectionName'),
    where('status', '==', 'active')
  );
  
  const querySnapshot = await getDocs(q);
  return querySnapshot.docs.map(doc => ({
    id: doc.id,
    ...doc.data()
  }));
}
```

### Query with Ordering and Limits

```typescript
import { collection, query, orderBy, limit, getDocs } from 'firebase/firestore';

async function getLatestItems(count: number = 10) {
  const q = query(
    collection(db, 'collectionName'),
    orderBy('createdAt', 'desc'),
    limit(count)
  );
  
  const querySnapshot = await getDocs(q);
  return querySnapshot.docs.map(doc => ({
    id: doc.id,
    ...doc.data()
  }));
}
```

## Best Practices

1. **Singleton Pattern:** The Firebase app is initialized only once using `getApps().length === 0` check

2. **Environment Variables:** All config values come from environment variables, never hardcode credentials

3. **Client-Side Only:** Firebase operations happen on the client side with `NEXT_PUBLIC_` prefixed variables

4. **Error Handling:** Always wrap async Firestore operations in try-catch blocks

5. **Security Rules:** Set up proper Firestore security rules in the Firebase Console

6. **Cleanup Listeners:** Always unsubscribe from real-time listeners when components unmount

7. **Type Safety:** Use TypeScript interfaces for Firestore documents

## Common Patterns

### Custom Hook for Firestore Collection

```typescript
import { useEffect, useState } from 'react';
import { collection, getDocs } from 'firebase/firestore';
import { db } from '@/lib/firebase';

export function useFirestoreCollection<T>(collectionName: string) {
  const [data, setData] = useState<T[]>([]);
  const [loading, setLoading] = useState(true);
  const [error, setError] = useState<Error | null>(null);

  useEffect(() => {
    const fetchData = async () => {
      try {
        const querySnapshot = await getDocs(collection(db, collectionName));
        const items = querySnapshot.docs.map(doc => ({
          id: doc.id,
          ...doc.data()
        })) as T[];
        setData(items);
      } catch (err) {
        setError(err as Error);
      } finally {
        setLoading(false);
      }
    };

    fetchData();
  }, [collectionName]);

  return { data, loading, error };
}
```

Usage:

```typescript
function MyComponent() {
  const { data, loading, error } = useFirestoreCollection<BlogPost>('blogPosts');
  
  if (loading) return <div>Loading...</div>;
  if (error) return <div>Error: {error.message}</div>;
  
  return (
    <div>
      {data.map(post => (
        <div key={post.id}>{post.title}</div>
      ))}
    </div>
  );
}
```
