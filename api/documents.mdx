# Document Management

The admin document endpoints enable uploading, managing, and processing documents for the RAG system.

## Endpoints Overview

| Method | Endpoint | Description |
|--------|----------|-------------|
| POST | `/admin/documents` | Upload and process a document |
| GET | `/admin/documents` | List all documents with chunk counts |
| GET | `/admin/documents/{id}` | Get specific document details |
| DELETE | `/admin/documents/{id}` | Delete document and all chunks |

## Upload Document

### POST /admin/documents

Upload a document for RAG processing. The system will:

1. Extract text from the file (PDF, TXT, or MD)
2. Split text into chunks (1000 chars, 200 overlap)
3. Generate embeddings for each chunk
4. Store everything in the database

<CodeGroup>
```bash cURL
curl -X POST http://localhost:8000/admin/documents \
  -F "file=@pranav_resume.pdf"
```

```python Python
import requests

with open("pranav_resume.pdf", "rb") as f:
    files = {"file": f}
    response = requests.post(
        "http://localhost:8000/admin/documents",
        files=files
    )
    
data = response.json()
print(f"Document ID: {data['document']['id']}")
print(f"Chunks created: {data['chunks_created']}")
```

```javascript JavaScript (with FormData)
const formData = new FormData();
formData.append('file', fileInput.files[0]);

fetch('http://localhost:8000/admin/documents', {

  });
```
</CodeGroup>

## List Documents

## Delete Document

### DELETE /admin/documents/{document_id}

Delete a document and all its associated chunks. This operation:

- Deletes the document record
- Automatically deletes all chunks (CASCADE)
- Is permanent and cannot be undone

<CodeGroup>
```bash cURL
curl -X DELETE http://localhost:8000/admin/documents/a1b2c3d4-e5f6-7890-abcd-ef1234567890
```

```python Python
import requests

doc_id = "a1b2c3d4-e5f6-7890-abcd-ef1234567890"
response = requests.delete(f"http://localhost:8000/admin/documents/{doc_id}")
data = response.json()

print(f"Deleted {data['chunks_deleted']} chunks")
```

```javascript JavaScript
const docId = 'a1b2c3d4-e5f6-7890-abcd-ef1234567890';

fetch(`http://localhost:8000/admin/documents/${docId}`, {
  method: 'DELETE'
})
  .then(res => res.json())
  .then(data => {
    console.log(data.message);
    console.log(`Deleted ${data.chunks_deleted} chunks`);
  });
```
</CodeGroup>

### Path Parameters

<ParamField path="document_id" type="uuid" required>
  UUID of the document to delete.
  
  **Example**: `a1b2c3d4-e5f6-7890-abcd-ef1234567890`

### Response

## Document Processing

### Text Extraction

From `src/routes/admin.py:50`:

#### PDF Files

```python
def extract_text_from_pdf(file_content: bytes) -> str:
    """Extract text from PDF file."""
    pdf = PdfReader(io.BytesIO(file_content))
    text = ""
    for page in pdf.pages:
        text += page.extract_text() or ""
    return text
```

- Uses pypdf library
- Extracts text from all pages
- Concatenates page content

#### Plain Text and Markdown

```python
def extract_text_from_plain(file_content: bytes) -> str:
    """Extract text from plain text or markdown file."""
    return file_content.decode("utf-8")
```

- Simple UTF-8 decoding
- Works for .txt and .md files

### Chunking Strategy

From `src/embeddings.py:32`:

```python
def chunk_text(text: str, chunk_size: int = 1000, chunk_overlap: int = 200) -> List[str]:
    """
    Split text into overlapping chunks.
    
    - Chunk size: 1000 characters
    - Overlap: 200 characters
    - Tries to break at paragraph (\n\n) or sentence boundaries
    """
```

**Algorithm**:

1. If text is less than or equal to 1000 chars, return as single chunk
2. Otherwise, split into chunks:
   - Target size: 1000 characters
   - Look for paragraph break (`\n\n`) in second half of chunk
   - If no paragraph break, look for sentence break (`. `, `! `, `? `, `\n`)
   - If no good break point, split at 1000 chars
   - Overlap 200 characters with next chunk

### Embedding Generation

From `src/embeddings.py:25`:

```python
def generate_embeddings(texts: List[str]) -> List[List[float]]:
    """Generate embeddings for multiple texts (batch processing)."""
    model = get_embedding_model()  # all-MiniLM-L6-v2
    embeddings = model.encode(texts, convert_to_numpy=True, show_progress_bar=True)
    return embeddings.tolist()
```

- Uses sentence-transformers all-MiniLM-L6-v2
- Batch processes all chunks at once
- Shows progress bar during processing
- Returns 384-dimensional vectors

### Storage

From `src/routes/admin.py:133`:

```python
# Create chunk records
chunk_records = []
for idx, (chunk_text_content, embedding) in enumerate(zip(chunks, embeddings)):
    chunk = Chunk(
        document_id=document.id,
        content=chunk_text_content,
        embedding=embedding,
        chunk_index=idx,
    )
    chunk_records.append(chunk)

db.add_all(chunk_records)
await db.commit()
```

- Stores each chunk with its embedding
- Maintains chunk order via `chunk_index`
- Links to parent document via `document_id`
- Commits all chunks in single transaction

## Performance Metrics

Typical processing times for a 10-page PDF resume:

- **Upload**: < 100ms
- **Text extraction**: ~200ms
- **Chunking**: ~10ms
- **Embedding generation**: ~500ms (12 chunks)
- **Database storage**: ~100ms
- **Total**: ~1 second

<Info>
  First run may be slower (~5 seconds) due to downloading the embedding model (~80MB).
</Info>
