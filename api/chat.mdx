# Chat Endpoint

The `/chat` endpoint enables RAG-powered conversations with context from uploaded documents.

## Endpoint

<CodeGroup>
```bash cURL
curl -X POST http://localhost:8000/chat \
  -H "Content-Type: application/json" \
  -d '{
    "message": "What are Pranav\"s skills in backend development?"
  }'
```

```python Python
import requests

response = requests.post(
    "http://localhost:8000/chat",
    json={
        "message": "What are Pranav's skills in backend development?"
    }
)

data = response.json()
print(data["response"])
for source in data["sources"]:
    print(f"Source: {source['document_name']} (similarity: {source['similarity']})")
```

```javascript JavaScript
fetch('http://localhost:8000/chat', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({
    message: "What are Pranav's skills in backend development?"
  })
})
  .then(res => res.json())
  .then(data => {
    console.log('Response:', data.response);
    console.log('Sources:', data.sources);
  });
```
</CodeGroup>

## POST /chat

Send a message and receive a RAG-powered response with source citations.

### Request Body

<ParamField path="message" type="string" required>
  The user's question or message.
  
  **Example**: `"What are Pranav's skills in backend development?"`
  
  <Warning>
    Message cannot be empty or whitespace-only.
  </Warning>
</ParamField>

<ParamField path="chat_history" type="array" default="null">
  Optional conversation history for context continuity.
  
  Array of message objects with `role` and `content` fields.
  
  **Example**:
  ```json
  [
    {"role": "user", "content": "What is Pranav's experience?"},
    {"role": "assistant", "content": "Pranav has 5 years of experience..."}
  ]
  ```
  
  <Note>
    Only the last 5 messages from chat history are used to maintain context window limits.
  </Note>
</ParamField>

#### Message Object Schema

<ResponseField name="role" type="string" required>
  Role of the message sender: `"user"` or `"assistant"`
</ResponseField>

<ResponseField name="content" type="string" required>
  Content of the message
</ResponseField>

### Response

<ResponseField name="response" type="string">
  The AI-generated response based on retrieved context.
  
  Generated using Azure OpenAI GPT-4.1 with:
  - Retrieved document context
  - System prompt for Pranav AI
  - Conversation history (if provided)
  - Temperature: 0.7
  - Max tokens: 1000
</ResponseField>

<ResponseField name="sources" type="array">
  Array of source chunks used to generate the response.
  
  Each source includes:
  - Document ID and name
  - Relevant content snippet
  - Similarity score (0-1)
  
  Only chunks with similarity > 0.3 are included.
</ResponseField>

#### Source Chunk Schema

<ResponseField name="document_id" type="uuid">
  UUID of the source document
</ResponseField>

<ResponseField name="document_name" type="string">
  Filename of the source document
  
  **Example**: `"pranav_resume.pdf"`
</ResponseField>

<ResponseField name="content" type="string">
  Content snippet from the chunk (truncated to 200 characters)
  
  Shows the relevant text that contributed to the response.
</ResponseField>

<ResponseField name="similarity" type="float">
  Cosine similarity score between query and chunk (0-1)
  
  Rounded to 3 decimal places. Higher scores indicate more relevant content.
  
  **Example**: `0.847`
</ResponseField>

## How It Works

The chat endpoint follows this RAG (Retrieval Augmented Generation) flow:

### 1. Embedding Generation

```python
# Generate embedding for user query
query_embedding = generate_embedding(request.message)
# Uses sentence-transformers all-MiniLM-L6-v2 model
# Returns 384-dimensional vector
```

### 2. Vector Similarity Search

```sql
SELECT 
    c.id,
    c.document_id,
    c.content,
    d.filename,
    1 - (c.embedding <=> '[0.1,0.2,...]'::vector) as similarity
FROM chunks c
JOIN documents d ON c.document_id = d.id
ORDER BY c.embedding <=> '[0.1,0.2,...]'::vector
LIMIT 5
```

- Uses pgvector's cosine similarity operator `<=>`
- Retrieves top 5 most similar chunks
- Joins with documents table for metadata

### 3. Context Filtering

```python
if similarity > 0.3:  # Only include relevant chunks
    context_parts.append(f"[From: {filename}]\n{content}")
```

- Filters chunks with similarity > 0.3
- Adds source attribution to each chunk
- Builds combined context string

### 4. LLM Generation

System prompt used:

```text
You are Pranav AI, a helpful assistant that answers questions about Pranav 
based on the provided context.

**Rules:**
- Answer questions based ONLY on the provided context
- If the context doesn't contain relevant information, say "I don't have 
  information about that in my knowledge base"
- Be concise and direct in your responses
- Use a friendly, professional tone
- When information comes from different documents, clearly distinguish and 
  attribute information to each source
- Always mention which document the information came from when relevant

**Context:**
{context}
```

Request to Azure OpenAI:

```python
response = await client.chat.completions.create(
    model="gpt-4.1",
    messages=[
        {"role": "system", "content": SYSTEM_PROMPT.format(context=context)},
        *chat_history[-5:],  # Last 5 messages
        {"role": "user", "content": query}
    ],
    temperature=0.7,
    max_tokens=1000
)
```

## Performance Considerations

- **Embedding Generation**: ~10-50ms locally with all-MiniLM-L6-v2
- **Vector Search**: ~5-20ms with pgvector indexes
- **LLM Generation**: ~500-2000ms depending on Azure OpenAI latency
- **Total Response Time**: ~1-3 seconds typical
