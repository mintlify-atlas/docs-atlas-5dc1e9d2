# Embeddings System

Learn how Pranav AI generates vector embeddings for semantic search.

## What are Embeddings?

Embeddings are vector representations of text that capture semantic meaning:

```text
Text: "Python programming language"

Embedding: [0.123, -0.456, 0.789, -0.234, ...] (384 dimensions)
```

**Key Properties**:

- **Similar text** → **Similar vectors** (close in vector space)
- **Different text** → **Different vectors** (far in vector space)
- **Semantic understanding**: Captures meaning, not just keywords

### Example

```python
# These have high similarity (same meaning)
embedding("Python developer")     # [0.12, 0.45, ...]
embedding("Python programmer")    # [0.13, 0.47, ...]  # Similar!

# These have low similarity (different meaning)
embedding("Python developer")     # [0.12, 0.45, ...]
embedding("snake species")        # [-0.67, 0.23, ...] # Different!
```

## Embedding Generation

### Single Text

From `src/embeddings.py:18`:

```python
def generate_embedding(text: str) -> List[float]:
    """Generate embedding for a single text."""
    model = get_embedding_model()
    embedding = model.encode(text, convert_to_numpy=True)
    return embedding.tolist()
```

**Usage**:

```python
from src.embeddings import generate_embedding

query = "What are Pranav's Python skills?"
embedding = generate_embedding(query)

print(len(embedding))  # 384
print(embedding[:5])   # [0.123, -0.456, 0.789, -0.234, 0.567]
```

**Performance**: ~10-50ms on CPU

### Batch Processing

From `src/embeddings.py:25`:

```python
def generate_embeddings(texts: List[str]) -> List[List[float]]:
    """Generate embeddings for multiple texts (batch processing)."""
    model = get_embedding_model()
    embeddings = model.encode(
        texts, 
        convert_to_numpy=True, 
        show_progress_bar=True
    )
    return embeddings.tolist()
```

**Usage**:

```python
from src.embeddings import generate_embeddings

chunks = [
    "Pranav is a Python developer...",
    "He has experience with FastAPI...",
    "Built RAG systems using pgvector..."
]

embeddings = generate_embeddings(chunks)

print(len(embeddings))      # 3
print(len(embeddings[0]))   # 384
```

**Performance**: ~500ms for 12 chunks

**Why batch processing?**

```python
# Good: Batch processing
embeddings = generate_embeddings(chunks)  # ~500ms for 12 chunks

# Bad: One at a time
embeddings = []
for chunk in chunks:
    embedding = generate_embedding(chunk)  # ~50ms × 12 = 600ms
```

Batch processing is ~10-20% faster due to:
- Model parallelization
- Reduced Python overhead
- Better GPU utilization (if available)

## Text Chunking

### Implementation

From `src/embeddings.py:32`:

```python
def chunk_text(
    text: str, 
    chunk_size: int = 1000, 
    chunk_overlap: int = 200
) -> List[str]:
    """
    Split text into overlapping chunks.
    
    Args:
        text: Text to split
        chunk_size: Maximum characters per chunk (default: 1000)
        chunk_overlap: Overlap between chunks (default: 200)
    
    Returns:
        List of text chunks
    """
```

### Algorithm

The chunking algorithm tries to break at natural boundaries:

<Steps>
  <Step title="Check if text fits in one chunk">
    ```python
    if len(text) <= chunk_size:
        return [text]
    ```
    
    If text is small enough, return as-is.
  </Step>
  
  <Step title="Look for paragraph break">
    ```python
    para_break = text.rfind('\n\n', start, end)
    if para_break > start + chunk_size // 2:
        end = para_break + 2
    ```
    
    Try to break at paragraph boundaries first (double newline).
  </Step>
  
  <Step title="Look for sentence break">
    ```python
    for sep in ['. ', '! ', '? ', '\n']:
        sent_break = text.rfind(sep, start, end)
        if sent_break > start + chunk_size // 2:
            end = sent_break + len(sep)
            break
    ```
    
    If no paragraph break, look for sentence endings.
  </Step>
  
  <Step title="Split at chunk_size">
    ```python
    chunk = text[start:end].strip()
    if chunk:
        chunks.append(chunk)
    ```
    
    If no good break point, split at chunk_size.
  </Step>
  
  <Step title="Add overlap">
    ```python
    start = end - chunk_overlap
    ```
    
    Next chunk starts 200 characters before end of current chunk.
  </Step>
</Steps>

### Example

**Input text** (1500 characters):

```text
Pranav is a software engineer with 5 years of experience. He specializes 
in backend development using Python and Go.

He has built several RAG systems using Azure OpenAI and pgvector. His 
experience includes document processing, embedding generation, and vector 
search optimization.

In his spare time, he contributes to open source projects and writes 
technical blog posts about AI and machine learning.
```

**Output chunks** (1000 chars, 200 overlap):

```python
chunks = [
    # Chunk 0 (0-1000)
    "Pranav is a software engineer...optimization.\n\n",
    
    # Chunk 1 (800-1500) - starts 200 before previous end
    "optimization.\n\nIn his spare time...machine learning."
]
```

### Why Overlap?

Overlapping chunks prevents information loss:

**Without overlap**:
```text
Chunk 1: "...experience with FastAPI and"
Chunk 2: "Django for building REST APIs..."
```

**With overlap**:
```text
Chunk 1: "...experience with FastAPI and Django for..."
Chunk 2: "...FastAPI and Django for building REST APIs..."
```

Both chunks contain the full phrase "FastAPI and Django for building REST APIs".

```python
from src.embeddings import generate_embedding
import numpy as np

def cosine_similarity(a, b):
    return np.dot(a, b) / (np.linalg.norm(a) * np.linalg.norm(b))

# Test semantic similarity
text1 = "Python backend developer"
text2 = "Python backend programmer"

emb1 = generate_embedding(text1)
emb2 = generate_embedding(text2)
```

To use a different model:

**1. Update configuration**:

```bash .env
EMBEDDING_MODEL=all-mpnet-base-v2
EMBEDDING_DIMENSION=768
```

**2. Update database schema**:

```sql
-- Modify vector column dimension
ALTER TABLE chunks 
ALTER COLUMN embedding TYPE vector(768);
```

**3. Regenerate embeddings**:

```python
# Reprocess all existing documents
for document in documents:
    chunks = chunk_text(document.content)
    embeddings = generate_embeddings(chunks)
    # Update database...
```

<Warning>
  Changing models requires regenerating all embeddings! Old embeddings won't be compatible.
</Warning>

    ```python
    from torch.utils.data import DataLoader
    
    # Create training examples
    examples = [

        InputExample(texts=['query', 'negative doc'], label=0.0),

        epochs=1
    ```
